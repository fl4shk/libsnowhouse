Notes on Changes (August 2025):
* `sltu`/`slts`
	* Support versions of these two with an immediate value argument
		* `sltu` immediate should be `uint16_t`
	* This change requires changing the GCC backend.
* `or` immediate:
	* change immediate value argument to `uint16_t`
* pc-relative branches (including `bl`)
	* increase size of immediates for branches (including `bl`)
	* left shift the immediate value by 2
	* these two changes need to be supported in the Binutils port.
* add full product instructions and 64-bit div/mod
	* This change requires changing the GCC backend.
* Interrupts stuff:
	* support both "to" and "from" versions of `cpy` 
	* `ie` will be store inside of `ids`
* Possible idea:
	* Support 64-bit add/sub/sltu/slts/shifts
	* These could use register pairs.
